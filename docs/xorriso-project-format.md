# Формат файла `.xorriso-project`

## Обзор

`.xorriso-project` — проприетарный формат файла проекта приложения **xorriso-ui**. Представляет собой JSON-файл с расширением `.xorriso-project`, описывающий конфигурацию записи оптического диска: список файлов для записи, параметры файловой системы ISO и опции записи.

Формат не является частью xorriso или какого-либо стандарта — он разработан специально для xorriso-ui.

**Кодировка:** UTF-8
**Формат:** JSON с отступом 2 пробела
**Права доступа при сохранении:** 0644
**Текущая версия формата:** 1

## Назначение

Файл проекта позволяет:

- **Сохранить** подготовленную структуру диска на потом — не нужно заново собирать список файлов
- **Поделиться** проектом — передать файл другому пользователю, чтобы он записал такой же диск
- **Повторить** запись — открыть проект и записать ещё один диск с теми же файлами и настройками
- **Хранить настройки** — параметры файловой системы и записи сохраняются вместе с проектом

## Структура верхнего уровня

| Поле | Тип | Описание |
|------|-----|----------|
| `version` | number (uint8) | Версия формата файла. Текущая версия: **1** |
| `name` | string | Название проекта (отображается во вкладке) |
| `filePath` | string | Абсолютный путь к файлу проекта на диске |
| `volumeId` | string | Идентификатор тома ISO (макс. 32 символа, латиница) |
| `entries` | FileEntry[] | Список файлов и папок для записи |
| `isoOptions` | ISOOptions | Параметры создания ISO-образа |
| `burnOptions` | BurnOptions | Параметры записи на физический диск |
| `createdAt` | string (ISO 8601) | Дата и время создания проекта |
| `updatedAt` | string (ISO 8601) | Дата и время последнего сохранения |

## FileEntry — элемент проекта

Каждый файл или папка, добавленный в проект, представлен записью `FileEntry`.

| Поле | Тип | Описание |
|------|-----|----------|
| `sourcePath` | string | Абсолютный путь к исходному файлу/папке в файловой системе хоста |
| `destPath` | string | Путь на будущем диске (например, `/Documents/report.pdf`) |
| `name` | string | Имя файла или папки |
| `isDir` | boolean | `true` — папка, `false` — файл |
| `size` | number | Размер в байтах (для папок — 0 или суммарный) |
| `modTime` | number | Время изменения, Unix timestamp в миллисекундах |

### Особенности

- При добавлении папки в проект записываются **рекурсивно** все вложенные файлы и подпапки — каждый как отдельная запись `FileEntry`
- `sourcePath` указывает на реальный файл в системе — если файл перемещён или удалён, запись на диск завершится ошибкой
- `destPath` определяет расположение файла в структуре ISO-образа, начинается с `/`

## ISOOptions — параметры файловой системы

Определяют, какая файловая система и расширения будут использованы при создании ISO-образа.

| Поле | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `udf` | boolean | `true` | UDF (Universal Disk Format) — обязательна для Blu-ray/BDXL, рекомендуется для DVD |
| `isoLevel` | number | `3` | Уровень ISO 9660: 1 (8.3 имена), 2 (31 символ), 3 (без ограничения размера файла) |
| `rockRidge` | boolean | `true` | Rock Ridge — POSIX-расширение: длинные имена, симлинки, права доступа. Важно для Linux |
| `joliet` | boolean | `true` | Joliet — расширение Microsoft: Unicode-имена до 64 символов. Для совместимости с Windows |
| `hfsPlus` | boolean | `false` | HFS+ — расширение Apple: совместимость с macOS |
| `zisofs` | boolean | `false` | Прозрачное сжатие данных в ISO (zlib). Уменьшает размер образа, но совместимость ниже |
| `md5` | boolean | `true` | Вычисление и запись контрольных сумм MD5 для верификации целостности данных |
| `backupMode` | boolean | `false` | Режим резервного копирования — сохраняет ACL, xattr и другие расширенные атрибуты файлов |

### Рекомендации по выбору

| Носитель | Рекомендуемые опции |
|----------|-------------------|
| CD-R/RW (для Linux) | `rockRidge: true` |
| DVD для обмена (Linux + Windows) | `rockRidge: true, joliet: true` |
| Blu-ray / BDXL | `udf: true` (обязательно) |
| Бэкап системы | `rockRidge: true, backupMode: true, md5: true` |
| Универсальный диск | `udf: true, rockRidge: true, joliet: true, isoLevel: 3` |

## BurnOptions — параметры записи

Определяют поведение при записи ISO-образа на физический диск.

| Поле | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `speed` | string | `"auto"` | Скорость записи: `"auto"`, `"1x"`, `"2x"`, `"4x"`, `"8x"`, `"16x"` и т.д. |
| `dummyMode` | boolean | `false` | Тестовый прогон — симуляция записи без фактической записи на диск |
| `verify` | boolean | `true` | Верификация записанных данных после завершения записи |
| `closeDisc` | boolean | `false` | Закрыть диск (финализировать) — дальнейшая запись невозможна |
| `streamRecording` | boolean | `false` | Stream Recording — потоковая запись для Blu-ray, повышает производительность |
| `eject` | boolean | `true` | Автоматически извлечь диск после завершения записи |
| `burnMode` | string | `"auto"` | Режим записи: `"auto"`, `"DAO"` (Disc-At-Once), `"TAO"` (Track-At-Once) |
| `padding` | number | `300` | Количество секторов отступа в конце записи (для совместимости) |
| `multisession` | boolean | `false` | Мультисессия — позволяет дописывать данные на диск позднее |

### Пояснения

- **`dummyMode`** полезен для проверки, что запись пройдёт без ошибок, не расходуя диск
- **`closeDisc`** и **`multisession`** взаимоисключающи: закрытый диск нельзя дописать
- **`burnMode: "DAO"`** записывает весь диск за один проход — лучше совместимость, но нет мультисессии
- **`padding`** в 300 секторов (~600 КБ) — стандартная практика для избежания проблем чтения в конце диска

## Жизненный цикл проекта

### Создание

1. Пользователь создаёт новый проект в интерфейсе (меню или кнопка)
2. Проект получает имя, Volume ID и значения по умолчанию для ISO/Burn опций
3. В этот момент файл на диске **не создаётся** — проект существует только в памяти
4. `filePath` пуст до первого сохранения

### Редактирование

1. Пользователь добавляет файлы и папки из файлового менеджера
2. Каждый добавленный элемент становится записью `FileEntry` с указанием `sourcePath` → `destPath`
3. Пользователь настраивает параметры ISO и записи
4. Все изменения отслеживаются флагом `modified`

### Сохранение

1. **Первое сохранение** — пользователь выбирает путь и имя файла (`SaveAs`)
2. **Последующие сохранения** — перезаписывают файл по существующему `filePath`
3. При сохранении автоматически обновляется поле `updatedAt`
4. JSON форматируется с отступом 2 пробела для читаемости

### Открытие

1. Пользователь выбирает файл `.xorriso-project` через диалог открытия
2. JSON парсится и загружается в новую вкладку
3. Поле `filePath` устанавливается из пути открытого файла

### Запись диска

1. Проект передаётся сервису записи, который:
   - Строит ISO в памяти xorriso из `entries` с параметрами `isoOptions` и записывает на диск с параметрами `burnOptions`
   - Опционально верифицирует записанные данные

## Версионирование

Поле `version` — целое число (uint8, диапазон 0–255), определяющее версию формата файла.

| Версия | Описание |
|--------|----------|
| 0 | Файлы, созданные до введения версионирования (поле отсутствует в JSON) |
| 1 | Текущая версия. Добавлено поле `version` |

При изменении структуры формата (добавление/удаление/переименование полей) версия должна быть увеличена, а изменения задокументированы в этой таблице.

Приложение должно корректно открывать файлы с `version: 0` (или без поля `version`) для обратной совместимости.

## Ограничения

- **Не хранит содержимое файлов** — только ссылки на пути в файловой системе (`sourcePath`). Если исходные файлы перемещены или удалены, запись невозможна
- **Volume ID** — максимум 32 символа, только ASCII (ограничение ISO 9660)
- **Абсолютные пути** — `sourcePath` и `filePath` содержат абсолютные пути, привязанные к конкретной системе
- **Только Linux** — формат и приложение работают только в Linux
